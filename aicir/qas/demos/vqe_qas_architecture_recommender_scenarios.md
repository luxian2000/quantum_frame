# VQE/QAS Supernet Architecture Search Scenarios

This demo describes problem scenarios for architecture search. The final
architecture must be generated by the existing `aicir.qas.vqe_loop` supernet
path, not by hand-authored rules in the recommender.

Do not hand-author the final architecture. For every scenario below, run
`python -m aicir.qas.vqe_loop` with `--initial-labels 0` and
`--supernet-native-count >= --k-min`; then read the generated architecture from
completed benchmark rows.

## Common Output Contract

The search output is a directory containing:

- `supernet_bootstrap_queue.csv`: supernet cheap-ranked top candidates. Read
  `architecture_id`, `ansatz_gene`, `canonical_arch_hash`,
  `supernet_rank_score`, `screening_energy`, and `supernet_init_params_ref`.
- `benchmark_table_<n>q_v2.csv`: fair VQE labels for bootstrap candidates.
  Use `fair_best_energy` and `delta_ref` for final comparison.
- `round1/round1_queue.csv`: next fair-label batch if `--rounds 1`.
- `round1/round1_benchmark_table.csv`: final completed labels after oracle
  expansion. Sort completed rows by `fair_best_energy`.

`screening_energy_is_final_label=false` is part of the contract. Screening
energy is only a cheap ranking signal.

To make the recommender TXT print the searched architecture directly, add the
completed vqe-loop output directory to the input JSON:

```json
{
  "result_dirs": ["outputs/my_completed_supernet_run"]
}
```

The TXT report prints `Top Architectures` first and copies `ansatz_gene` from
those result directories. If `result_dirs` is absent or empty, the TXT only
prints the search jobs to run next.

## Scenario 1: Ising TFIM Chain

**Problem:** Estimate the ground state of a transverse-field Ising chain.

**Input shape:**

```json
{
  "scenario": "Ising TFIM Chain",
  "problem": "Ground-state search for a transverse-field Ising chain.",
  "hamiltonian": {
    "kind": "pauli_terms",
    "terms": [[-1.0, "ZZII"], [-1.0, "IZZI"], [-0.5, "XIII"], [-0.5, "IXII"]]
  },
  "top_k": 3
}
```

**Search command shape:**

```bash
python -m aicir.qas.vqe_loop tfim_terms.json \
  --initial-labels 0 \
  --rounds 1 \
  --k-min 3 \
  --supernet-native-count 3 \
  --supernet-native-layers 2 \
  --supernet-native-supernet-num 1 \
  --supernet-native-steps 80 \
  --supernet-native-ranking-num 40 \
  --supernet-native-finetune-steps 80 \
  --backend numpy \
  --dtype complex128
```

**Architecture source:** completed rows in `benchmark_table_<n>q_v2.csv` and
`round1/round1_benchmark_table.csv`.

## Scenario 2: Hamiltonian Path / Cycle QUBO

**Problem:** Encode a Hamiltonian path or Hamiltonian cycle QUBO as a diagonal
Ising Hamiltonian and search for a low-energy state-preparation ansatz.

**Input shape:** generate Pauli terms from
`demos/VQA_QAS_demo_hamiltonian_path.py` or
`demos/VQA_QAS_demo_hamiltonian_cycle.py`.

**Search command shape:**

```bash
python -m aicir.qas.vqe_loop hamiltonian_path_9q.json \
  --initial-labels 0 \
  --rounds 1 \
  --k-min 3 \
  --supernet-native-count 3 \
  --supernet-native-layers 2 \
  --supernet-native-supernet-num 1 \
  --supernet-native-steps 150 \
  --supernet-native-ranking-num 30 \
  --supernet-native-finetune-steps 250 \
  --backend numpy \
  --dtype complex128
```

**Architecture source:** the supernet-native `ansatz_gene` in the completed
`trackB_supernet` rows. For these diagonal Ising demos, the native 9q default
keeps the large-scale single-qubit gate pool compatible with the demo supernet
path.

## Scenario 3: H2 Molecule

**Problem:** Small molecular ground-state benchmark for H2.

**Input shape:**

```json
{
  "scenario": "H2 Molecule",
  "problem": "Ground-state VQE/QAS sanity check.",
  "hamiltonian": {
    "preset": "h2_sto3g_jw_r0735_4q"
  },
  "top_k": 3
}
```

**Search command shape:**

```bash
python -m aicir.qas.vqe_loop h2_input.json \
  --initial-labels 0 \
  --rounds 1 \
  --k-min 3 \
  --supernet-native-count 3 \
  --supernet-native-layers 2 \
  --supernet-native-supernet-num 1 \
  --supernet-native-steps 80 \
  --supernet-native-ranking-num 40 \
  --supernet-native-finetune-steps 80 \
  --backend numpy \
  --dtype complex128
```

**Architecture source:** sort completed benchmark rows by `fair_best_energy`;
read the selected architecture from `ansatz_gene`.

## Scenario 4: LiH Molecule

**Problem:** Larger molecular ground-state task where fair labels are expensive.

**Input shape:**

```json
{
  "scenario": "LiH Molecule",
  "problem": "Find a lower-energy VQE/QAS architecture for LiH.",
  "molecular_spec": {
    "molecule": "LiH",
    "distance": 1.5,
    "basis": "sto3g",
    "mapping": "jordan_wigner",
    "driver": "pyscf",
    "estimated_n_qubits": 12,
    "estimated_terms": 631
  },
  "top_k": 3
}
```

**Search command shape:**

```bash
python -m aicir.qas.vqe_loop lih_input.json \
  --initial-labels 0 \
  --rounds 1 \
  --k-min 3 \
  --supernet-native-count 3 \
  --supernet-native-layers 2 \
  --supernet-native-supernet-num 1 \
  --supernet-native-steps 150 \
  --supernet-native-ranking-num 80 \
  --supernet-native-finetune-steps 150 \
  --backend numpy \
  --dtype complex128
```

**Architecture source:** `trackB_supernet` completed rows first; if no
improvement appears, widen with `--supernet-native-count 5`,
`--supernet-native-supernet-num 2`, and `--supernet-native-ranking-num 160`.

## Scenario 5: Generic Pauli Hamiltonian

**Problem:** User has weighted Pauli terms that may or may not come from
chemistry.

**Input shape:**

```json
{
  "scenario": "Generic Pauli Hamiltonian",
  "problem": "Search an ansatz for a supplied Pauli Hamiltonian.",
  "hamiltonian": {
    "kind": "pauli_terms",
    "terms": [[-0.5, "ZI"], [0.25, "IZ"], [0.1, "XX"]]
  },
  "top_k": 3
}
```

**Search command shape:**

```bash
python -m aicir.qas.vqe_loop pauli_input.json \
  --initial-labels 0 \
  --rounds 1 \
  --k-min 3 \
  --supernet-native-count 3 \
  --supernet-native-layers 2 \
  --supernet-native-supernet-num 1 \
  --supernet-native-steps 100 \
  --supernet-native-ranking-num 60 \
  --supernet-native-finetune-steps 100 \
  --backend numpy \
  --dtype complex128
```

**Architecture source:** choose the completed row with the lowest
`fair_best_energy`; then copy its `ansatz_gene` into follow-up validation runs.

## Implementation Notes

The recommender demo may generate scenario JSON and TXT search instructions.
It must not invent the final ansatz. The final architecture is the serialized
`ansatz_gene` produced by the vqe-loop supernet-native path and validated by
fair VQE.
